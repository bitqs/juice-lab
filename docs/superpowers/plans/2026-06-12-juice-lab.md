# Juice Lab Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 单页交互 demo,10+1 步逐层叠加打击感技术,全部参数溯源自 vs-anatomy/hades-anatomy/Obsidian 笔记。

**Architecture:** vanilla JS + Canvas 2D,零依赖。juice 技术 = 独立模块实现统一钩子接口(onHit/onKill/onUpdate/onDraw),enabled 标志自由开关;步进状态机只是"打开前 N 个模块"的预设。

**Tech Stack:** HTML/CSS/JS(ES modules),WebAudio 合成音效,无构建步骤。

**Verification:** 视觉项目,按用户研究笔记《截图在环验证AI产出》:每任务完成后 playwright/chrome-devtools 截图人眼核对,不写单元测试(纯函数除外)。

- Create: `js/main.js`(rAF 循环、输入、JuiceRegistry、StepMachine)
- Create: `js/scene.js`(实体与战斗)

**接口契约(后续任务全部依赖,签名必须一致):**

```js
// scene.js
export const scene = {
  player: { x, y, w, h, facing, attackT },   // attackT: 0=idle, 0..1 攻击动画进度
  dummy:  { x, y, w, h, hp, maxHp, alive, respawnT, flashT, vx, vy, tint },
  attack(),                       // 触发一次攻击(受 juice9 タメツメ改变时序)
  update(dt), draw(ctx),
}
// 命中时 scene 调用: juice.fire('onHit', {target, dmg, crit, kill, dir})
// 击杀时:           juice.fire('onKill', {target, dir})

// main.js
export const juice = {
  modules: [],                    // 注册顺序 = 步骤顺序
  register(mod),                  // mod: {id, name, enabled, onHit?, onKill?, onUpdate?, onDraw?, drawOver?}
  fire(hook, payload),            // 只调 enabled 模块
}
// 全局时间控制(顿帧用): main 维护 timeScale 与 freezeUntil
```
