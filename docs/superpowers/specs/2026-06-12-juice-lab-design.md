# Juice Lab(打击感实验室)设计文档

2026-06-12 · 已经用户确认(方案 A:独立项目)

## 目标

一个交互式单页 web,让人**亲手感受**打击感(game feel / juice)是由哪些技术叠出来的:
从第 0 步"毫无手感"开始,每步叠加一层技术,随时开关对比。
所有技术与参数溯源自 vs-anatomy 反编译笔记、hades-anatomy、Obsidian 游戏设计笔记。

三重用途:① 打击感研究的可玩展示;② 自媒体支柱 3 传播物料(每步一条 GIF);③ 三国幸存者调参参考。

## 形态与栈

- vanilla JS + Canvas 2D,零依赖,单页(同 thegametheory 栈)
- 桌面:鼠标点击/空格攻击;移动:触摸攻击
- 自动连击模式(录 GIF / 解放双手观察)
- 中文 UI
- 仓库 `bitqs/juice-lab`(public),Cloudflare Pages 部署

## 场景

左侧 Canvas:玩家方块(剑士抽象)+ 训练假人(会自动复活,死亡有尸体弹飞)。
点击/按键 → 玩家挥击 → 命中假人 → 当前开启的 juice 层依次表达。
伤害随机区间,~12% 暴击(暴击走更重的反馈档)。

## 10+1 步阶梯

| 步 | 技术 | 关键参数(溯源) |
|---|---|---|
| 0 | 裸命中 | 对照组:血条扣血,无任何反馈 |
| 1 | 闪白 tintFill | 120ms 纯白,死亡缩短 60ms(VS HITVFX) |
| 2 | 顿帧 hitstop | 普攻 0 / 暴击 55ms / 处决(击杀)80ms(VS + 樱井递增律) |
| 3 | 击退 + 死亡弹飞 | 速度取反;尸体沿攻击向量弹飞(VS deathKB) |
| 4 | 伤害数字 | ±0.15 X 随机偏移,弹跳缓动 上→落→淡出(VS Blitter) |
| 5 | 命中贴片 + 冲击波 | 接触点星形 + 扩散环(VS HITVFX 表) |
| 6 | 死亡粒子爆裂 | 8-16 粒,放射状,重力+淡出 |
| 7 | 屏震 | **普攻零屏震**,仅击杀/暴击触发(VS 稀缺资源原则) |
| 8 | 音效分层 | WebAudio 合成:命中(中频短)/暴击(低频重,Vlambeer)/击杀(下扫) |
| 9 | タメツメ前摇 | 攻击动画慢蓄→快斩不对称配速(打击帧笔记) |
| 10 | 全开 + 倒U警告 | 附"过量"滑杆演示 juice 过载降可读性(CHI 2024) |

## 界面

- 左:Canvas 场景(正方形,自适应)
- 右:当前步说明卡 — 技术名 / 真实参数 / 为什么有效 / 出处(笔记名)
- 底:← 上一步 | 下一步 → | 步进度点;自由模式开关面板(每层独立 checkbox,任意组合)
- 顶:第 0 步 ⇄ 当前配置 一键 A/B 对比按钮(传播主物料)
- GIF 按钮:MediaRecorder 录 Canvas 5 秒(v1 先实现按钮+提示,录制可后补)

## 架构

```
index.html        — 布局骨架
style.css         — 深色像素风(对齐 thegametheory 气质)
js/main.js        — 游戏循环(rAF)、输入、步进状态机
js/scene.js       — 玩家/假人实体、攻击判定、血条、复活
js/juice.js       — 10 个技术模块,统一接口 {id, name, apply hooks},全部可独立开关
js/steps.js       — 步骤数据(说明文案、参数、出处)
js/audio.js       — WebAudio 合成音效(零音频文件)
```

juice 模块通过钩子接入:onHit(target, dmg, crit) / onKill / onUpdate(dt) / onDraw(ctx)。
每个技术 = 一个独立对象,enabled 标志控制 — 自由组合零耦合。

## 不做(YAGNI)

- 不做多敌人/波次/成长系统(那是三国幸存者的事)
- 不做美术资源(全部代码绘制,呼应"手感靠参数化而非美术帧")
- 不做英文版(反响好再说)
- v1 不做 GIF 实录,占位即可

## 成功标准

- 第 0 步手感"死鱼",第 10 步"上头",每步差异肉眼可感
- 每步参数可溯源到具体笔记/反编译事实
- 手机可玩
- Lighthouse 性能无明显问题(单 Canvas,60fps)
